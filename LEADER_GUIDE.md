## 一、投屏模式 —— 让聚会开始前多一份温度

### 它是什么

聚会开始前，你可以把今日团契成员的**心境词云**投放到大屏或白板上。屏幕上只会出现"感恩""平安""疲惫"这样的状态词，以及每个状态有几位弟兄姐妹，没有任何人名、内容或头像。

这不是点名，而是一起安静片刻，让在场的每个人感受到：**今天我们带着什么来到这里**。

### 如何开启

1. 登录后，点击底部导航进入**团契**页面。
2. 在页面右上角，点击"**组长后台**"按钮（或访问 `/fellowship/console`）。
3. 后台页面的右上角有一个 **「投屏」** 按钮，点击后会在新标签页打开投屏页面。
4. 将浏览器切换到**全屏模式**（通常按 `F11` 或从浏览器菜单选"全屏"），再投到投影仪或大屏上即可。

```
组长后台（右上角）
       │
       └──▶  「投屏」按钮（Monitor 图标）
                  │
                  └──▶  新标签页打开全屏词云
                         /fellowship/console/projector
```

### 几件小事

- 投屏页面**每 30 秒自动刷新**，随着组员陆续提交当日内室，词云会实时更新，不需要手动刷新。
- 如果今日还没有人提交，屏幕会显示"今日尚无分享"，这是正常的，可以先播着，等人到齐。
- 投屏页面没有任何导航栏，纯粹全屏展示，适合投影仪环境。

## 二、牧养授权 —— 关怀从不越界

### 背景：为什么要"授权"才能看到名字

当有组员在提交内室时勾选了"需要代祷"，系统会向你发出一个**匿名信号**。这个设计是刻意的——

> 组员愿意举手求助，不代表愿意立刻让组长知道是自己。
> 只有当组员主动点击"同意关怀"，你才能看到对方的名字。

这是系统对每位组员的承诺，也是你在使用牧养功能时需要了解的边界。

### 完整流程

#### 第一步：你看到匿名信号

进入**组长后台**（`/fellowship/console`），在"代祷关怀"区域，你会看到一张橙色卡片：

```
┌──────────────────────────────────────────────┐
│  有组员标记了代祷需求          [请求关怀]      │
│  2026年5月15日                               │
└──────────────────────────────────────────────┘
```

卡片上没有名字、没有状态，只有时间。你知道"有人需要关怀"，但不知道是谁。

#### 第二步：你选择"请求关怀"

点击卡片右边的 **「请求关怀」** 按钮。

这一步的意思是：**"我看到了这个需求，我愿意主动去关怀。"**

系统会向那位组员发出通知——告知有人希望来关怀他/她。此时，卡片变成"等待成员回应…"状态，你仍然看不到名字。

#### 第三步：组员自行决定

那位组员会在自己的**今日内室**页面看到一条通知，内容大致是"你的组长希望与你进一步交流，是否同意？"

组员可以：
- 点击 **「同意」** → 授权给你，你随后可以看到对方名字
- 点击 **「暂不」** → 婉拒，卡片显示"已婉拒关怀"，无需追问

#### 第四步：授权后你才看到名字

当组员点击同意后，卡片更新为绿色，显示对方的名字，并标注"已授权"。

```
┌──────────────────────────────────────────────┐
│  小明                              ✓ 已授权   │  ← 绿色
│  2026年5月15日 发出关怀请求                   │
└──────────────────────────────────────────────┘
```

接下来，如何关怀、何时联系，完全交由你和对方的真实关系来推进，系统不会再介入。

### 状态对照表

| 你看到的 | 含义 | 此时可以做什么 |
|---|---|---|
| 橙色匿名卡片 | 有人举手，尚未联系 | 点击"请求关怀"，向对方发出邀请 |
| 等待成员回应… | 已发出邀请，等待对方决定 | 耐心等候，不要追问 |
| ✓ 已授权 + 名字 | 对方同意，可以正式关怀 | 按照你的方式去联系 |
| 已婉拒关怀 | 对方暂时不希望被关怀 | 尊重这个决定，继续在祷告中记念 |

## 三、AI 词云 —— 它知道什么，不知道什么

### AI 看到的原始材料

每位组员提交内室时，AI 收到的只有两样东西：
1. 组员选择的**心境标签**（感恩 / 平安 / 疲惫 / 干渴 / 混乱）
2. 录音转成的**文字内容**（最长 800 字）

AI 生成一段简短的安慰话语和一节经文，还会提炼一份不超过 140 字的"交托摘要"——这份摘要用第三人称写成，**系统指令明确要求 AI 不得包含任何可识别个人信息、不得出现姓名、地名或机构名**。

### AI 摘要的加密保存

这份摘要在服务器端**立即加密**（AES-256-GCM），以密文形式存入数据库，组长和管理员都无法直接读取。

只有当组员自己提交当日内室后，团契页才会解密展示其他成员的摘要——作为一种"先分享，再聆听"的团契默契。

### 你在组长后台看到的 AI 洞察

进入组长后台，有一张金色背景的"**AI 氛围预备**"卡片，显示"本周团契建议"。

这份建议的原始材料是：近 3 日全体组员的**状态标签分布数字**——例如"疲惫 4 人、平安 2 人、干渴 1 人"。

AI **看不到任何一个人说了什么**，它只接收到这些统计数字，然后基于这个整体氛围，给出一段聚会方向的建议，比如适合做什么主题的分享、需要预留更多安静的时间，或者这周可以主要以感恩为主调。

你可以点击卡片右上角的**刷新图标**随时更新建议。

### 投屏词云与 AI 洞察的隐私对比

| | 投屏词云 | AI 氛围建议 |
|---|---|---|
| 数据来源 | 今日状态标签（点名计数） | 近 3 日状态分布数字 |
| 有没有摘要内容 | ❌ 没有 | ❌ 没有 |
| 有没有人名 | ❌ 没有 | ❌ 没有 |
| 谁能看到 | 聚会现场所有人（投屏） | 仅你（组长） |
| 刷新频率 | 每 30 秒自动刷新 | 手动点击刷新 |

## 常见问题

**Q：组员提交"静默入账"，我能看到他们的内容吗？**

不能。静默入账不会触发 AI，也没有任何摘要。你只会在词云上看到"平安"多了一个人，但不知道是谁，也没有任何文字内容。

**Q：投屏模式需要组员先提交才有内容显示吗？**

是的。词云只显示当日已提交内室的组员状态。如果聚会开始时还没人提交，屏幕会显示"今日尚无分享"，可以先投着，随着组员陆续到场提交，词云会自动更新。

**Q：组员可以不让我看到他们的摘要吗？**

可以。组员提交时可以勾选"静默入账"，这样 AI 不会生成摘要，词云上也只是匿名计一个数，对其他组员完全不可见。

**Q：我发出"请求关怀"后，对方多久会看到通知？**

对方下次打开今日内室页面时，顶部会看到通知。系统不会发短信或推送，请在生活中也温柔地留意对方。

**Q：如果对方一直没有回应，我该怎么办？**

卡片会一直停留在"等待成员回应…"的状态，不会自动过期。这个等待本身就是一种尊重。你可以在祷告中记念，或者在合适的时机以自然的方式问候，不需要在系统内催促。
